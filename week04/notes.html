Chapter 8 - Forms

  • Semantic markup is very important for both maintainability and accessibility.
  • The <form> tag takes an action attribute referencing the web page to link to when the form is submitted.
  • The document.forms property is a list of all the forms in the document.
  • document.forms is ordered by which form appears first in the HTML.
  • Each form in document.forms can be accessed by its name attribute.
  • Just like id, the name of an element should be unique.
  • Forms typically contain buttons and inputs.
  • The submit button sends the form to the server.
  • The reset button resets the form.
  • Reset buttons probably shouldn't be used.
  • Forms have events too.
  • The focus event is fired when the user is paying attention to the form.
  • The blur event is fired when the user is no longer paying attention ot the form.
  • The change event is fired when the user changes the information in an element of the form and moves on.
  • The submit event is fired when the user clicks the submit button in the form.
  • Sanity checks can be performed when handling the submit event.
  • The placeholder attribute gives little prompts about what information should be input in a given element of a form.
  • The autofocus attribute of a form's child element can be used to automatically focus on that element for better productivity.
  • input fields can be of type password, which makes the value unreadable.
  • Input fields can be of type checkbox, which represents a boolean value input.
  • There are a lot of other input types.
  • I could take notes on all the input types but that would be an unproductive use of time.
