    • transform: inherit;
    • transform: initial;

    • transform: none;
    • transform: perspective(10px);
    • transform: revert;
    • transform: rotate( 45deg);
    • transform: rotateX( 45deg);
    • transform: rotateY( 45deg);
    • transform: rotateZ( 45deg);
    • transform: scale(1.5);

  • Transitions allow the smoothe appearance of changing state.
  • transition-delay sets the time to wait before causing the changes.
  • transition-timing sets the time it takes to complete the changes
  • transition-property specifies the properties that transition.
  • transition-timing-function specifies the mathematical function that describes how the property changes from one value to another.
  • transition-timing-function may be any of these values:
    • transition-timing-function:cubic-bezier;
    • transition-timing-function:ease;
    • transition-timing-function:ease-in;
    • transition-timing-function:ease-in-out;
    • transition-timing-function:ease-out;
    • transition-timing-function:inherit;
    • transition-timing-function:initial;
    • transition-timing-function:jump-both;
    • transition-timing-function:jump-end;
    • transition-timing-function:jump-none;
    • transition-timing-function:jump-start;
    • transition-timing-function:linear;
    • transition-timing-function:revert;
    • transition-timing-function:step-end;
    • transition-timing-function:step-start;
    • transition-timing-function:steps;
    • transition-timing-function:unset;

  • Let's be honest: Canvas was made to replace Flash.
  • Canvas element coordinate system specified by widht and height attributes of elements.
  • Canvas can have 2D context or 3D webgl context
  • 2D context draws and fills geometry with vector drawing commands common to every other language.
  • WebGL context has API very similar to OpenGL ES.
  • W3C has a working group for the WebGPU API designed to be as powerful as the almost-universal Vulkan API, Microsoft's DirectX 12 API, and Apple's metal API.
  • Canvas can draw other image elements, like other canvases, videos, images.
  • CORS trust applies to canvas drawing and image manipulation
  • It is possible to directly manipulate every pixel in a canvas element.
  • canvas animation can be better timed with animation frames.
  • SVG is the XHTML of vector graphics
  • How do you remember that the namespace uri is "http://www.w3.org/2000/svg"?
  • SVG reminds me of postscript.
  • Instead of specifying an exact algorithm to draw an image, describe what the image looks like.
  • Geometry can get very complex.
  • There are a lot of programs to make SVG images so you don't have to.
  • SVG animation is possible.
  • w3c specification for SVG
  • SVG scripting and DOM manipulation is possible depending on security and rendering level.
