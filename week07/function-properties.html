Function Properties

Chrome says these are the properties of any function:

  • length
  • name
  • apply
  • arguments
  • bind
  • call
  • caller
  • constructor
  • toString
  • hasOwnProperty
  • isPrototypeOf
  • propertyIsEnumerable
  • toLocaleString
  • valueOf
  • __defineGetter__
  • __defineSetter__
  • __lookupGetter__
  • __lookupSetter__
  • __proto__

I will go over what I can find out about these, one at a time:

  • length

    This refers to the arity of the function.

  • name

    This is the name of the variable that holds the function. Lambdas have an empty function string. I think it exists for debugging purposes.

  • apply

    This calls the function. It can be useful if I want to iterate through a list of functions.

  • arguments

    Chrome throws an error when I try to call this function.

  • bind

    Returns a function that calls the given function after setting the "this" attribute and other hard-coded values. It's a nice way to give a function generic values.

  • call

    Calls the function with a specified this and arguments. Whereas call takes a variable number of arguments,apply takes an array containing the arguments.

  • caller

    Chrome throws an error when I try to call this function.

  • constructor

    MDN says this returns the function that created an instance of an object. Functions are objects too.

  • toString

    Returns a string description of an object for the debugger to use.

  • hasOwnProperty

    The ECMAScript 2020 draft says this tests if an object has a property corresponding to a given key.

  • isPrototypeOf

    The ECMAScript 2020 draft says this function tests if this is in the given object's hierarchy.

I stopped here because it became clear that the remaining properties belong to Object, not String.
