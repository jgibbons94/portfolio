Doubloon Crossers

  • This has been a big project of mine for the past several weeks.
  • Two clients exist:
    • Display client or "central client"
    • Input client or "user client"
  • Both clients are static web pages.
  • Communication is accomplished with web sockets.
  • Web sockets server is in Racket.
  • Web Sockets server is written to die after 5 minutes without activity. This is resolved by having the clients send a ping every few minutes.
  • Messages are sent between server and clients in JSON format.
  • "Message" tells the recipient what to do, interpreted manually.
  • "params" are the parameters to the message.
  • I used higher order functions to handle messages like events.
