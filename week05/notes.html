READING

  • error = bad information
  • exception = "I detected an error"
  • throw and try/catch exceptions.
  • Stack traces show where an exception was thrown.
  • Warning means there could be something wrong. Usually there's been some recent refactoring.
  • Strict Mode means potential errors will be caught earlier.
  • Modules automatically are in strict mode.
  • Linting tools enforce a style guide.
  • Try to detect features before using them.
  • Nobody likes alert. Don't use it. Use console functions instead.
  • See console api documentation here.
  • You can throw anything, but the best practice is to throw error objects and exceptions instead of strings and numbers and objects.
