READING

  • BOM lets programs work with browser and monitor.
  • BOM has no official standard according to outdated book. Standard is actually described in HTML standard according to MDN.
  • All global variables declared outside of modules belong to the window object.
  • Outdated book shows off legacy code, including alert, prompt, and confirm
  • navigator contains the userAgent value, which lets code discriminate by browser.
  • navigator also lets scripts identify available media capabilities, media sessions, mimetypes, permissions, platform, plugins, and other information that lets scripts adjust to capabilities.
  • navigator can be used to fingerprint, so there are security-oriented browsers and extensions designed to break expected conventions.
  • location contains information about the current URL and methods to change the current URL.
  • history lets links navigate through browser history.
  • history contains the number of pages the current tab has been to
  • Data provided by history can be used with server logs to determine if the user was browsing in the current tab before navigating to the current page.
  • window.open can create a popup window, which is very annoying.
  • I think most modern browsers just make window.open make a new tab, but I haven't tested it.
  • Windows made from window.open can be closed in a script. This is still annoying.
  • Don't use window.open, window.move, or window.resize if you want people to visit your site.
  • window.screen contains fingerprint data for the user's screen.
  • I think window.screen has been replaced by css @media rules, because there's little legitimate use for window.screen outside styling.
  • Cookies are data sent to the server and stored in the browser.
  • Cookies are mostly replaced by the more secure sessionStorage and localStorage which don't transfer data with html.
  • Cookies are still used for authentication.
  • Cookies expire, get old, and crumble, but browser crashes may prevent this.
  • Cookies set properties with strings. I think it's because they predate JSON.
  • setTimeout runs a function after the specified number of milliseconds.
  • setInterval runs a function every specified number of milliseconds.
  • setInterval might be used for animation.
  • requestAnimationFrame is a better animation call because it calls the function on the next rendering frame.
  • Timing functions return an id that can be used to cancel them.
  • CSS is better for DOM animations than JS.
  • requestAnimationFrame is mostly useful for the canvas api.
  • Lists of HTML/JS/CSS APIs, links to their specifications, known bugs, and statistics about their availability can be found at html5 test and can i use
  • These references are more up-to-date than the book.
  • Attributes beginning with data- are officially supported.
  • data- attributes can have any name that begins with data- and meets the other xml tag requirements.
  • data- attributes are for embedding any type of data.
  • I've been using localStorage since week 1 of this course. sessionStorage is identical except it's deleted at the end of a session.
  • Geolocation gives a program the user's location.
  • I've seen it used for getting prices listed at local franchise establishments.
  • I suppose it can also be used for an augmented reality mechanic like pokemon go.
  • Web Workers are the response to JS being single threaded but websites needing heavy calculations.
  • I don't think they're relevant anymore with async functions.
  • The WebSockets API is a way to communicate with a server when the server thinks it's ready. It cuts out significant overhead and replaces XHRs in some instances like chatrooms.
  • The Notifications API is a way to send notifications from a page running in the background to the desktop.
  • WebSockets and Notifications are often seen combined.
  • Multimedia can be embedded with the <audio> and <video> tags.
  • Multiple sources can be provided for a piece of media with <source> tags.
  • The Web Audio API allows a programmer to run calculations on audio samples or generate audio streams.
  • The GamePad API gets input from gamepads (game controllers) connected to the client.
  • I used the GamePad API when I submitted a game to the Experimental Gameplay Project. I think that was back in 2012 or 2013. The resulting game wasn't particularly good, but I think I was able to effectively utilize an xbox 360 controller.
  • I do not know the best way to handle outdated clients. Polyfills can be helpful I suppose, but they add latency when the client is up-to-date. I don't think it's worth supporting browsers more than 3 years old.
